Написать простое веб‑приложение, которое загружает персонажей из открытого API и отображает их в виде карточек.

## Техническое задание

1. **Использовать Vite**  
   - Проект должен быть собран на Vite.
   - Должен работать в режиме разработки (`npm run dev`) и продакшена (`npm run build`).

2. **Разделить логику на модули**  
   - `fetcher.js` – модуль с загрузкой данных.
   - `ui.js` – модуль с логикой отрисовки в DOM.
   - `main.js` – точка входа, связывает загрузку и отрисовку.

3. **Класс‑подобный подход (через объекты и стрелочные функции)**  
   - Создать объект‑“fetcher” с методом `getData(url)`, который:
     - использует `fetch`;
     - асинхронно возвращает данные;
     - обрабатывает ошибки через `try...catch`.
   - Создать объект‑“UI” с методами:
     - `render(data)` – принимает массив объектов и отрисовывает карточки персонажей;
     - `clear()` – очищает контейнер перед новой загрузкой.

4. **Логика работы приложения**  
   - При загрузке страницы или по клику на кнопку:
     - создаются экземпляры “fetcher” и “UI”;
     - сначала вызывается загрузка данных;
     - затем вызывается отрисовка.
   - Перед повторной загрузкой список должен полностью очищаться.

5. **Использовать API**  
   Бесплатное API:  
   https://rickandmortyapi.com/api/character  
   - Выводить имя, статус, вид и количество эпизодов у каждого персонажа.
   - Показывать изображение персонажа.

## Как это реализовано

- В `src/fetcher.js` есть стрелочная функция `getData`, которая делает `fetch`, обрабатывает ошибки и возвращает массив персонажей.
- В `src/ui.js` функция `createUI(containerId)` возвращает объект с методами `render(data)` и `clear()`, чтобы отрисовывать карточки и чистить контейнер.
- В `src/main.js` создаются объекты `fetcher` и `ui`, и логика связана так: сначала `getData`, затем `ui.render`.

Такой подход позволяет:
- легко менять источник данных (API),
- переиспользовать UI‑логику в других проектах,
- не дублировать код по загрузке и отрисовке.
